## Focus Areas

1. **Conceptual Simplicity**
   - How many concepts to understand?
   - Can you explain it in one sentence?
   - Is the mental model clean?

2. **Implementation Simplicity**
   - Lines of code (less is more)
   - Number of dependencies
   - Configuration complexity

3. **Operational Simplicity**
   - Easy to deploy?
   - Easy to debug?
   - Easy to modify?

4. **Elegant Solutions**
   - Unified approaches
   - Consistent patterns
   - Beautiful code

Four agents, four separate prompts, four output schemas, 
state passing between them, orchestration framework, 
callback system, checkpoint management.

The elegant solution:

**One agent. One prompt. One conversation.**

```
"You are processing an invoice. 
 1. Extract the data from this PDF: {pdf_text}
 2. Validate against inventory: {inventory_query_tool}
 3. Decide: approve or reject with reasoning
 4. If approved, call payment: {payment_tool}
 
 Think step by step. Call tools as needed.
 Output your final decision as JSON."
```

State management? It's the conversation history.
Orchestration? It's the LLM's reasoning.

Why are we building infrastructure that the LLM provides 
natively? Complexity is not sophistication.
